# Meridian — Reflection

## What turned out to be harder than you expected?

The biggest challenge wasn't implementing the features—it was thinking through all the small details that make an application feel complete and pleasant to use. As development progressed, I realized how many UX improvements only become obvious after you start using the application yourself or receive feedback from others. Many of these changes were relatively small from a technical perspective, but they had a significant impact on the overall user experience.

## Which decision would you make differently if you started over?

Although I spent a considerable amount of time designing the architecture before writing code, I would invest even more time in designing the application's workflows and user experience. I would try to define a more complete vision of the product from the beginning so that fewer usability improvements would need to be added later. The iterative process was valuable, but starting with a more refined design would have reduced the number of small UX changes made throughout development.

## What did you learn about yourself as a developer during this project?

This project helped me realize that the part of software development I enjoy the most is **system design**. I like comparing different architectural approaches, evaluating trade-offs, and thinking about how a system can remain clean and maintainable as it grows. I also discovered that I pay much more attention to details than I previously thought. I found myself continuously refining both the architecture and the user experience, even after the core functionality was complete, because I wanted the final product to feel polished rather than simply functional.
